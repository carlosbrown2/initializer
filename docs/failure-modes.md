# Failure-mode register

Every module, function, and data flow that can fail must appear here, paired with a mechanical check that catches the failure before merge. See `project-kickoff-prompt.md` §1 for the contract.

**Status values:** `covered` | `proven-impossible` | `out-of-scope`

**Categories:** `correctness` | `concurrency` | `atomicity` | `input` | `resource` | `temporal` | `version` | `dependency` | `operational` | `security`

| Module / function | Failure mode | Category | Check (file:test) | Status |
|-------------------|--------------|----------|-------------------|--------|
| scripts/ralph/ralph.sh gate-result parsing | agent emits `<gate-result>PASS</gate-result>` without running the gate (Goodhart / gate-bypass) | correctness | scripts/hooks/install.sh pre-push hook re-runs the gate command from CLAUDE.md and blocks push on non-zero exit; divergence against scripts/ralph/ralph.sh-persisted .last-gate-result is reported in the block message | covered |
| scripts/hooks/parsers.sh register-integrity parsers | edge cases (escaped pipes, Unicode, trailing whitespace, 5-column rows, multi-line continuation rows) silently accept malformed rows or reject valid ones; drift between the inlined pre-commit awk and the callable parsers | correctness | tests/hooks/parsers.bats exercises every parser against known-good and known-bad fixtures plus smoke tests against the real registers; scripts/hooks/install.sh sources parsers.sh so there is only one implementation | covered |
| docs/skills/review-rubric.md (Phase 1 completion) | unedited starter rubric ships with the project, letting "Review verdict" be claimed `bounded` while severity classification is still anchored only to generic clauses (Goodhart). A naive guard that only matches the disclaimer phrase admits stub rubrics with no clauses and rubrics that copy the starter clauses verbatim | correctness | scripts/hooks/parsers.sh rubric_edit_check rejects rubrics that (a) still contain the starter disclaimer, (b) keep the starter H1 header verbatim, (c) define no clauses at all, or (d) define only clauses already in the starter allowlist; scripts/hooks/install.sh wires it into the pre-commit hook once any bead is in_progress; tests/hooks/parsers.bats exercises all four reject paths plus the accept path; Phase 1 bootstrap (no in-progress bead) is exempt | covered |
| CLAUDE.md verification gate command | gate is structurally a no-op — trailing `\|\| <always-succeed>` (`\|\| true`, `\|\| :`, `\|\| 0`, `\|\| exit 0`, etc.) binds to the whole `&&` chain so any in-chain failure (real syntax error, future check) silently exits 0; or a non-bash file (markdown) is added to a `bash -n` chain forcing a soft-fail escape. The pre-push hook re-runs whatever the gate is, so a no-op gate makes the hook a no-op too | correctness | tests/hooks/gate.bats asserts the extracted gate command has no soft-fail-escape tail (`\|\| true`, `\|\| :`, `\|\| 0`, `\|\| exit 0`), contains no `.md` paths, passes against the current repo, and propagates non-zero exit when any in-chain clause is corrupted | covered |
| tests/hooks/gate.bats "gate passes against the current repo" sanity test | the gate now invokes `bats tests/hooks/`, which re-invokes this very test file. Without a recursion guard the sanity test would fork-bomb the gate invocation into infinite nested bats runs | correctness | tests/hooks/gate.bats sentinel `_BATS_GATE_REENTRY`: outer invocation exports the flag before running the gate; the nested bats run sees the flag and skips the sanity test so only the outer run exercises the full chain against the real repo | covered |
| scripts/hooks/parsers.sh syntax errors | a syntax error in parsers.sh silently breaks every register-integrity check (install.sh sources it at runtime); without a gate clause, the failure surfaces only when a developer hits pre-commit on an affected register edit | correctness | CLAUDE.md verification gate includes `bash -n scripts/hooks/parsers.sh`; tests/hooks/gate.bats "gate fails when scripts/hooks/parsers.sh has a syntax error" corrupts parsers.sh in a temp tree and asserts the chain exits non-zero, which also fail-closes a future accidental drop of the parsers.sh clause from the gate | covered |
| scripts/hooks/install.sh review artifact validator | validator only matched clause-shape (`P[123]\.[a-z][a-z-]*`) and did not check membership in the rubric, so a review artifact citing `P1.totally-made-up-clause` passed exit 0 — turning the "Review verdict" decision-register row into a Goodhart on clause-shape rather than on the actual canonical clause set | correctness | scripts/hooks/parsers.sh review_artifact_clauses_check extracts the canonical clause set from `**P[123].name**` bold-marker definitions in docs/skills/review-rubric.md and rejects any cited clause not in that set; scripts/hooks/install.sh wires it into the review-artifact validator so review-bead commits with invented clauses block; tests/hooks/parsers.bats covers reject (invented clause), accept (real clauses), accept-after-rubric-add (no separate registration), reject-when-rubric-empty, plain-prose-citation reject, and a smoke test that synthesizes an artifact citing every real clause from the shipped rubric | covered |
