# Decision register

Every place agent variance can enter this project must appear here, paired with the structural mechanism that bounds it. See `project-kickoff-prompt.md` §2 for the contract.

**Status values:** `bounded` | `ritual-bounded` | `agent-discretion` | `escalation-only`

| Decision point | Where it occurs | Bounding mechanism | Enforcement | Status |
|----------------|-----------------|--------------------|-------------|--------|
| Solution selection | impl bead execution | every acceptance criterion bound to a mechanical check | failure-mode register hook in scripts/hooks/install.sh | bounded |
| Acceptance interpretation | every "is it done" call | every PRD criterion expressible as test/type/proof | PRD review at Phase 1; re-validate on PRD edit | ritual-bounded |
| Review verdict | review beads | severity rubric in docs/skills/review-rubric.md; the review-bead contract in scripts/ralph/prompt.md mandates each finding cite a clause from that rubric, and the human reading the review artifact catches drift | review-bead contract in scripts/ralph/prompt.md; human read of docs/reviews/<bead-id>.md | ritual-bounded |
| Pattern extraction | compound beads | every promoted pattern carries a `model:` tag and a retire-on-upgrade rule. The "binding artifact citation" prose in CLAUDE.md's `## Discovered Patterns` intro is a ritual contract for compound beads, not a hook-enforced bound — pattern_citation_check was retired in agent-template-e5u under the harness-restraint principle | scripts/hooks/parsers.sh claude_model_tags_check, wired into scripts/hooks/install.sh's pre-commit chain when CLAUDE.md is staged; tests/hooks/parsers.bats covers accept (model: tag present), reject (no model: tag), section-bounding (ignores ### outside ## Discovered Patterns, closes on next ## heading), and a smoke test against the live CLAUDE.md | bounded |
| Decomposition | Phase 2 | bead schema: scope, deps, criteria, size, register DoD | beads CLI + human dep-graph review at Phase 2 | ritual-bounded |
| Tool / search choice | execution | unconstrained — model picks | none (rationale: search strategy is exactly where we want model freedom) | agent-discretion |
| Model upgrade drift | model swap | every promoted pattern tagged with source model; retire unless re-validated | upgrade ritual: re-run both registers under the new model before resuming | ritual-bounded |
| Scope creep | every commit | `.current-bead-scope` declares allowed paths for impl/pare/compound beads; review/research beads are read-only except for normal infrastructure and the current docs/reviews/<bead-id>.md artifact | scope enforcement and review/research write-protection in scripts/hooks/install.sh | bounded |
| Artifact format | review / research beads | review artifacts cite the rubric and contain a severity clause | review-bead contract in scripts/ralph/prompt.md; human read of docs/reviews/<bead-id>.md | ritual-bounded |
| Sampling variance | every model invocation | non-interactive agent mode (`claude --print`, `codex exec`, or equivalent), single-shot, fresh context per bead | scripts/ralph/ralph.sh invocation; one-bead-per-iteration | bounded |
| Confidence | post-agent routing on BEAD_DONE | single-axis: bash-observed gate result is the routing input — PASS → HIGH, anything else → LOW. The prior 4-axis stack (diff size, touched_hooks, touched_claude_md with carve-out) was a hand-calibrated heuristic stacked on top of the gate verdict; the collapse lets the gate be the single source of truth | scripts/ralph/lib.sh compute_confidence is sourced by scripts/ralph/ralph.sh and called on BEAD_DONE with the gate result; scripts/ralph/lib.sh read_auto_land_policy / should_auto_land route the verdict against the `auto-land:` policy in CLAUDE.md; tests/hooks/ralph.bats covers gate-FAIL/SKIPPED/empty terminal LOW, gate-PASS HIGH, and trailing-positional-args silently ignored (pins the "ignore legacy N-arg shape" invariant against re-introduction) | bounded |
| Verification truth | every "done" claim | one command from CLAUDE.md, not agent judgment | scripts/ralph/ralph.sh runs the gate itself via scripts/ralph/lib.sh run_gate on BEAD_DONE and writes the real exit code to .last-gate-result (sourcing scripts/hooks/parsers.sh gate_command_extract); pre-push hook from scripts/hooks/install.sh re-runs the gate command on git push as defense-in-depth and blocks on divergence | bounded |
| Failure-mode coverage binding | implementation beads mark register rows as `covered` | `covered` rows must cite a check file in the Check column, and each cited check file must be named directly in the extracted verification gate or live under a gated directory token | scripts/hooks/parsers.sh fm_gate_refs_check, wired into scripts/hooks/install.sh's pre-commit chain; tests/hooks/parsers.bats covers direct-file accept, directory accept, absent-check reject, non-covered-row ignore, and real-register smoke coverage | bounded |
